# Compile error
> Compilation failed: Shouldn't be exposed: deserialized class Intermediate

Pre-requisites
1. You have an `abstract class` (say `Base) `with an `@Composable abstract fun`
2. You have another `abstract class` (say `Intermediate`) which extends the class in Step 1, but _does not override_ the abstract `@Composable fun`
3. The class in step 2 is somehow exported to iOS.

**Note:** If you create a concrete class that directly extends `Base`, then the error is not seen. It seems to occur only if you extend a class that has an `@Composable abstract fun` but does not override that function.

# Steps to repro

1. Clone this repo
2. Run the app on Android and iOS. It works fine (the app was created from [this template](https://terrakok.github.io/Compose-Multiplatform-Wizard/) and the functionality or UI is not important for the repro)
3. Open the file `issue.kt` in `composeApp/src/commonMain/lotlin` in the package `org.company.app`
4. Uncomment the 2 commented classes (`Intermediate` and `Concrete`)
5. Run the app on Android again. It runs fine.
6. Run the app on iOS. You see the compile error pasted in the stack trace below.
